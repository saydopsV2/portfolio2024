Projet de Graphes

  • Langage: Java
  • Date du projet: 27 mai 2024

Graphe des migrations aux États-Unis avant et après construction de faisceaux d'arêtes. Ce graphe contient 1090 sommets (villes) et 5635 arêtes (flux migratoires).

Le but de ce projet est de construire une application de visualisation de graphe.

La visualisation d'information est le sous-domaine de l'informatique qui consiste à définir et implémenter des représentations graphiques de données dans le but de permettre à une utilisatrice ou un utilisateur de les explorer efficacement. Ici les données d'intérêt seront modélisées par un graphe affiché sous la forme d'un diagramme nœud-lien. Dans ce type de diagramme, chaque nœud du graphe est représenté par une forme géométrique (p. ex., un cercle, un carré, etc.) et chaque arête par une ligne (ou courbe) reliant les formes géométriques correspondant à ses extrémités. Bien qu'intuitive, cette technique de représentation souffre rapidement de "bruit visuel" induit par le nombre important de croisements d'arêtes (mais aussi par le nombre de pixels utilisés pour chaque arête, le nombre de pixels de l'écran étant borné).

Afin d'améliorer la lisibilité de telles représentations, il existe deux solutions principales :

- affecter des coordonnées aux sommets afin de minimiser les croisements (et longueurs) d'arêtes et s'il n'est pas possible de déplacer les sommets (car leurs positions encodent une information), nous avons regroupé les arêtes en faisceaux afin de minimiser le nombre de pixels utilisés pour dessiner les arêtes mais aussi de mettre en évidence les flux principaux dans le réseau.

Dans ce projet, nous nous intéresserons à la deuxième approche, appelée edge bundling.
